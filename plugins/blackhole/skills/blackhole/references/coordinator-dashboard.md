# Coordinator Dashboard ‚Äî Main Chat Status

The coordinator is the user's **visibility layer** in Multitask Mode. The orchestrator
runs in background; the main chat must stay informative without regurgitating full
orchestrator turn logs.

## Command

```bash
bun run status
# or: bun run scripts/campaign-status.ts --campaign-dir .blackhole
```

Reads `.blackhole/config.json`, `queue.json`, `findings-ledger.json`,
`campaign-checkpoint.md`, and (when `gh` is available) scoped forge open counts.

## When to print (REQUIRED)

| Event | Who | Action |
|-------|-----|--------|
| Campaign start (before spawning orchestrator) | `coordinator` | Run `bun run status` ‚Üí print **full** output to user |
| Orchestrator background turn completes (idle notification) | `coordinator` | Run `bun run status` ‚Üí print **full** output, then resume orchestrator if work remains ‚Äî fires only when orchestrator has cleared its in-flight worker barrier and ended its turn |
| User asks `status` / `@blackhole status` | Coordinator or orchestrator | Run `bun run status` ‚Üí print full output; do not spawn workers |
| Intake files a GitHub issue | `coordinator` | Print one line: `üìã Filed #N ‚Äî <title> (milestone <M>)` then re-run status if campaign is active |
| Orchestrator ends turn | `orchestrator` | Ensure checkpoint written; coordinator prints dashboard on notification |

**Do not** collapse the dashboard to a one-line confirmation. Users rely on the main
chat for campaign overview.

## Dashboard sections

The status script emits markdown with:

1. **Header** ‚Äî scope (milestone/labels), orchestrator turn, queue `refreshed_at`
2. **Counts** ‚Äî forge open issues/PRs, queue active/done/in-flight/blocked/ready, ledger severities
3. **In-flight** ‚Äî table: issue, phase, PR, notes
4. **Blocked** ‚Äî issue list with blocker reason
5. **Ready** ‚Äî Pareto-ready issue numbers
6. **Completed** ‚Äî merged/closed in queue
7. **Issues filed** ‚Äî ledger rows with `deferred_to_issue` (discovery filings)
8. **Ledger open** ‚Äî top open V-code findings
9. **Active workers** ‚Äî from checkpoint `## In-flight workers` section

## Coordinator turn flow (with visibility)

```
Orchestrator turn completes (notification)
  ‚Üì
Coordinator: bun run status ‚Üí print full dashboard to user
  ‚Üì
If queue work remains and not blocked on user: resume orchestrator
  ‚Üì
END TURN
```

## Anti-patterns

- "Turn 5 complete" with no dashboard
- Resuming orchestrator without printing status when user has not seen progress
- Replacing dashboard with subagent `user_visible_high_level_summary` only ‚Äî summary is a teaser; dashboard is SSOT for chat
- Orchestrator ends turn with `## In-flight workers` populated ‚Äî campaign stalls; workers complete but no agent triages (`multitask-mode.md` ¬ß Cursor Pattern B ‚Äî Background worker barrier)
