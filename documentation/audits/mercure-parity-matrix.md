# Mercure Parity Matrix

Living, single-maintainer table of mercure enforcement mechanisms and their blackhole
equivalent. Seeded (M2 of the `mercure-parity-program` initiative, ADR-013 D1) from
`documentation/audits/mercure-parity-surface.md` — the verified evidence base and seed source
of record; citations below are not re-derived from the mercure plugin cache directly. `prj-mercure-sync`
is the sole future writer; a self-audit or reviewer finding a stale row files an issue, it does
not edit this file directly.

**Row-count footnote**: `mercure-parity-surface.md` §1b labels the diff-conditional checklist
bucket "(3)" in its prose but then names 4 items (Integration Coherence, Information Hierarchy,
Extension Tax, Doc Governance) — this matrix reproduces all 4 as distinct `checklist` rows
(PM-016..019) per `milestone-2.md`'s Touch-Paths T1 instruction to footnote, not silently
correct, the source doc's own label/count mismatch.

**GAP-6 / GAP-7 are process-level, not single mechanism-clusters**, so they are not given their
own row: **GAP-6** (x-security-audit exploitability-methodology depth vs blackhole
`V-SEC-06..10` — unconfirmed equivalence) is folded into PM-004 (Security Review)'s citation
below, flagged for verify-before-build per `mercure-parity-surface.md`'s Recommendations §3, not
resolved by this seed run. **GAP-7** (~65 mercure skill domains never swept) is the meta-problem
this entire matrix's `gap`/`N/A` rows collectively replace as the itemized successor to
`mercure-sync.md`'s retired "everything else" Coverage-table row (see this milestone's T2).
